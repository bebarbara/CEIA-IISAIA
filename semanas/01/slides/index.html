Naturaleza estocastica

  • El siguiente token predicho no siempre es el mismo
  • No necesariamente coincide con lo que estaba en el dataset
  • Distribucion de probabilidad sobre todo el vocabulario
Hacer hincapie en la naturaleza estocastica de las LLMs: no siempre el siguiente token que se predice es el mismo, ni tampoco necesariamente el que estaba en el dataset. Hay una distribucion de probabilidad sobre todos los posibles tokens y se muestrea de ahi. GPT-2 fue el primer modelo que demostro esto de forma convincente.

Alucinaciones

Por que alucinan las LLMs

  • En el dataset de entrenamiento, todas las preguntas se responden con confianza
  • El modelo no tiene acceso a internet, solo genera tokens
  • Responde con el mismo tono seguro, sepa o no la respuesta
En el dataset de entrenamiento todas las preguntas son respondidas con confianza, entonces todo lo que le preguntes lo va a tratar de responder manteniendo ese tono, mas alla de que sepa la respuesta o no. El modelo a esta altura no tiene acceso a internet, simplemente es un generador de tokens.

Reduccion de alucinaciones

Enfoque Llama 3

  • Descubrir informacion que el modelo no sabe
  • Armar dataset donde la respuesta correcta es "No lo se"
  • Proceso automatizable con IA

Paper de Llama 3

Hablar de como Meta disminuyo las alucinaciones para Llama 3. Basicamente, van descubriendo ejemplos de informacion que el modelo no sabe (agarrando articulos de Wikipedia y analizandolos con LLMs para descubrir que informacion no es dada en el articulo) y con eso arman un nuevo dataset donde la respuesta correcta es "No lo se". Este proceso puede ser realizado completamente por IA.

Busqueda como solucion

  • Dar al modelo la capacidad de buscar en internet
  • Tokens especiales para disparar busquedas
  • El conocimiento de pre-training permite que pocos ejemplos de post-training basten
Una forma de disminuir alucinaciones es utilizar mecanismos que permitan popular el contexto de la LLM con informacion relevante. Por ejemplo, dandole al modelo la capacidad de buscar en internet usando tokens especiales. La combinacion de conocimiento adquirido en pre-training hace que solo un par de ejemplos en post-training permita que la LLM sepa hacer buenas busquedas. Mostrar esto con la interfaz de ChatGPT.

Conocimiento embebido vs contexto

  • Conocimiento en los pesos del modelo: "vago recuerdo"
  • Informacion en el contexto: memoria funcional
El conocimiento embebido en los pesos del modelo es como un vago recuerdo, mientras que meterle informacion en el contexto es mas bien una memoria funcional. Por eso mecanismos como busqueda en internet o RAG son tan efectivos para reducir alucinaciones.

El "yo" de un modelo

  • No tiene sentido tratarlo como si tuviese un sentido del "yo"
  • Las respuestas sobre si mismo son una mezcla estocastica del dataset
  • Dataset hard-coded de identidad
No tiene sentido preguntarle al modelo preguntas sobre quien es o tratarlo como si tuviese un sentido del yo. Al final es un generador de tokens entrenado con texto de internet y calibrado para responder como asistente. Lo que responda a ese tipo de preguntas es una mezcla estocastica de todo eso combinado con lo que el proveedor del modelo le agregue por defecto al contexto. Mostrar el dataset de OLMo con ejemplos de preguntas y respuestas hard-coded sobre identidad. Mostrar diferencia entre decirle "who are you" a un modelo base vs un modelo instruct.

Generacion de tokens para pensar

  • Cada token usa la misma capacidad de computo
  • Generar tokens intermedios relevantes distribuye la capacidad de pensamiento
  • Respuestas con razonamiento previo son mejores
Generar intuicion acerca de como las LLMs necesitan generar tokens para pensar. Cada nuevo token que se genera va a usar la misma capacidad de computo sin importar su relevancia. Si en lugar de responder directamente, el modelo genera tokens relevantes para llegar a una respuesta, la respuesta final sera mejor porque distribuye su capacidad de pensamiento entre todos esos tokens. Usar Tiktokenizer como ayuda visual.

Tool use y aritmetica

  • Aritmetica y deletreo no son el punto fuerte de las LLMs
  • Decirle "use code" genera y ejecuta un programa
  • Descomponer problemas grandes en problemas chicos mejora resultados
Para resolver problemas de aritmetica, decirle al modelo "use code" va a hacer que genere y ejecute un programa que lo calcule. En general, usar metodos que permitan al modelo descomponer el problema en problemas mas chicos que puede ir resolviendo va a hacer que el resultado sea mejor. Explicar por que tareas como aritmetica o deletreo no son el punto fuerte: los tokens no se alinean con digitos o letras individuales.

Reinforcement Learning

Reinforcement Learning Post-Training

  • Ultima etapa del entrenamiento de una LLM moderna
  • Analogo a hacer los ejercicios de un libro
  • Prueba y error hasta llegar al resultado correcto
Ultima etapa del entrenamiento de una LLM moderna. Versiones de ChatGPT anteriores a o1, o3 no hacian esto. Es analogo a hacer los ejercicios de un libro: miras la letra del ejercicio, ves el resultado en el solucionario e iterativamente vas haciendo prueba y error hasta llegar al resultado. No siempre es claro que tipo de respuesta es mejor mostrarle a una LLM durante el entrenamiento, porque varias pueden parecer validas.

El modelo descubre como pensar

  • Darle un problema para resolver
  • Pedirle muchos intentos de solucion
  • Evaluar cual intento es mas correcto, corto y claro
  • Usar esa respuesta para entrenar el modelo
Lo que se quiere es que el modelo descubra por su cuenta como le sirve pensar. La forma de hacer eso es darle un problema para que resuelva, pedirle muchas veces que trate de resolverlo, evaluar en cual de esos intentos logro el resultado mas correcto (no solo correcto sino tambien el mas corto o mejor explicado) y usar esa respuesta generada para entrenar el modelo.

DeepSeek

  • Publico su paper a principios de 2025
  • Modelo de nivel similar a los de grandes empresas
  • Procedimiento de reinforcement learning publicado abiertamente
Antes de que DeepSeek publique su paper a principios de 2025, los conocimientos sobre como hacer reinforcement learning para LLMs eran cerrados. DeepSeek saco un modelo de nivel muy similar al de las grandes empresas e hizo publico el procedimiento que utilizo.

Thinking tokens y momentos eureka

  • Token <thinking> para encapsular razonamiento
  • El modelo descubrio por su cuenta que pensar largo le servia
  • Momentos de "eureka" durante el razonamiento
Lo interesante de DeepSeek es que introdujeron el uso del token thinking para encapsular los tokens generados por el modelo relacionados con pensar y no con la respuesta en si. El modelo descubrio por su propia cuenta, con ensayo y error, que la forma de generar tokens que mas le servia para resolver problemas era con pensamientos largos en los que va evaluando distintas formas de resolver el problema y teniendo momentos de eureka donde se va dando cuenta de que logro avances.

AlphaGo

  • Imitar a los mejores jugadores no es suficiente para superarlos
  • Es necesario que el modelo aprenda por si mismo
DeepMind al entrenar su modelo que juega al Go dejo muy en evidencia que solamente tratar de imitar a los mejores jugadores del mundo no era suficiente para ser mejor que ellos. Es necesario permitir que el modelo aprenda por si mismo para lograr eso. Esta es la misma idea detras del reinforcement learning en LLMs.
